# Vice scripts

- `build.sh` ‚Äî restore and build `Vice.slnx`.
- `test.sh` ‚Äî run all tests in `Vice.slnx`.
- `install-hooks.sh` ‚Äî point `core.hooksPath` at the tracked `.githooks/` directory so the committed `pre-push` hook runs `build.sh` and `test.sh` before every push, catching format/build/test failures locally. Run once per clone.
- `bench.sh` ‚Äî run the `Vice.Benchmarks` BenchmarkDotNet harness over the hot paths; runs every benchmark by default, or forwards BenchmarkDotNet arguments (e.g. `--filter '*RouteStrategy*'`). Pass `--gate` to run as a regression gate: the harness diffs each benchmark's mean against `bench/Vice.Benchmarks/baseline.json` and exits non-zero when any benchmark slows past the tolerance (`VICE_BENCH_TOLERANCE`, default `0.10` for 10%); a missing baseline is seeded from the current run. Pass `--update-baseline` to rewrite the baseline from the current run.
- `install-local.sh` ‚Äî pack `Vice.Cli` and `Vice.Mux.Cli` into `artifacts/local-nupkg/` and install them as the global `vice` and `vice-mux` tools through the repo-root `nuget.config`, whose `<packageSourceMapping>` binds the `Vice*` IDs to that local source. Uses `dotnet tool update --global`, an in-place replacement with no tool-absent window: a failed pack or install leaves the previously installed tool intact. Set `VICE_INSTALL_SYSTEMD=1` to also install the `packaging/systemd/vice-daemon.service` user unit (`Restart=on-failure`), which supervises `vice --daemon` so the accept loop is relaunched after an unrecoverable fault; enable it with `systemctl --user enable --now vice-daemon.service`.
- `uninstall-local.sh` ‚Äî uninstall the global `Vice.Cli` and `Vice.Mux.Cli` tools and remove the local nupkg cache.
- `release.sh` ‚Äî pack `Vice.Cli` and `Vice.Mux.Cli` at the `Directory.Build.props` `<Version>` into `artifacts/release/<version>/`. Dry-run by default; `--push` publishes the `.nupkg` files to `$VICE_NUGET_FEED` (default nuget.org) using `$NUGET_API_KEY`; `--verify-tag` requires an annotated git tag `v<version>` at HEAD before packing.
- `rollback.sh <version> [vice|vice-mux|all]` ‚Äî pin an installed tool to a prior published version via `dotnet tool update --global <pkg> --version <version>`. Roll back from a non-default feed by setting `$VICE_NUGET_FEED`.
- `demo.sh` ‚Äî build, install, exercise a few `vice` commands, then uninstall. Pass `--with-net` to include network-dependent samples.
