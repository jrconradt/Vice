# Localization and internationalization

## Scope decision: user-facing text is English-only

All user-facing strings emitted by `vice` and the Vice framework — help output, usage text, error messages, hints, and diagnostics — are authored in English and are not externalized into a resource catalog (`.resx`, `ResourceManager`, or a string table). This is a deliberate product decision for a developer-facing CLI, not an oversight.

There is no translation layer. Adding one would require externalizing every literal into a resource catalog; that work is out of scope and intentionally not begun.

## What `--locale` does (and does not) do

The `--locale` global option takes a BCP-47 tag (e.g. `en-US`, `de-DE`) and sets `CultureInfo.CurrentUICulture` for the duration of the command. It affects **culture-sensitive formatting** — numbers, dates, and sort order — only. It does **not** translate any message text.

`--locale de-DE` formats numbers and dates in the German convention; the surrounding words stay in English.

## If translation enters scope

Externalize the user-facing literals into a resource catalog before the surface grows further, and route message lookups through it. Until then, English is the single supported UI language.
